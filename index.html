BEGINNER exercise

Using the HTML5 validation attributes describe at the link below, implement the following validation rules for this form:
http://www.w3schools.com/html/html_form_attributes.asp

  • All inputs must have a placeholder value
  • All fields marked with an asterix must be required
  • Currency values must increment/decrement by 1c be between 0 and 199.99
  • The 'Title' must be between 3 and 200 characters, whereas all other text fields must be between 3 and 100 characters
  • Email addresses must be a valid email format
  • Dates must be a valid date format

INTERMEDIATE exercise

Apply the attribute 'novalidate' (http://www.w3schools.com/tags/att_form_novalidate.asp) to the form. This will disable the default HTML5 validation attributes.

Now, implement the exact same validation rules using vanilla JavaScript inside the script.js file (you can make use of the 'validateForm' method).

In this exercise you will need to research how to implement JavaScript Regular Expression (RegEx) validation. A general overview of RegEx is provided on wikipedia (https://en.wikipedia.org/wiki/Regular_expression). To REALLY understand what they are you'll need to replicate someone elses implementation - and adjust it for your own needs. For this I suggest you Google the phrase "javascript regex validation example". HINT: regular expression is required for the 'Date of publication' and 'Email' fields.

Output any errors or feedback in an alert window.
